# Contributing
Thinking about contributing? That's awesome! We really appreciate it!
Whether you're reporting a bug, suggesting an idea, improving documentation, or contributing code, it all helps move the project forward.
This guide explains how to get involved and how we keep things organized and respectful.

---

## Ways You Can Help
You can contribute by:

- Reporting bugs
- Suggesting features or improvements
- Improving documentation
- Writing, optimizing, or refactoring code
- Reviewing pull requests

---

## Getting Started
1. **Fork the repository**

2. **Clone your fork locally**

```bash
git clone https://github.com/[username]/[repo-name].git
cd [repo-name]
```

3. **Create a new branch**

```bash
git checkout -b feature/your-feature-name
```

4. Install dependencies and run the project locally
   (Check the `README.md` for specific setup instructions)

---

## Development Expectations
When contributing, please:
- Write clean, readable code
- Add or update tests when needed
- Update documentation if you change behavior

Before opening a pull request, make sure your branch is up to date with the latest `main`/`master` branch.

---

## Pull Requests
When you open a pull request:
- Clearly explain what you changed and why
- Link related issues (if any)
- Include screenshots or logs when helpful
- Make sure all checks are passing

We review pull requests based on:
- Correctness
- Readability and maintainability
- Test coverage
- Alignment with the project's direction

Reviews might take a while, so please be patient!

---

## Code Style
- Follow the existing structure and style
- Use any configured linters or formatters
- Avoid mixing unrelated changes in the same pull request

If we follow a specific style guide, it'll be listed in the `README.md` or project configs!

---

## Testing
- Add tests for new features or bug fixes when possible
- Make sure existing tests pass before submitting
- If automated testing isn't available, include clear manual testing steps

---

## Reporting Bugs
When reporting a bug, please include:

- A clear, descriptive title
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Environment details (OS, version, browser, game or webiste, etc.)

If there's a bug report template, please use it, it helps us respond faster!

---

## Suggesting Features
We love seeing our communities ideas! If you have an idea, let us know!
When suggesting a feature:
- Explain the problem you're trying to solve or the feature you're thinking of
- Describe your proposed solution
- Mention any alternatives you've considered

This helps us decide whether it fits the vision of the project.

---

## Documentation
Clear documentation makes everything easier.
You can help by:
- Fixing typos
- Clarifying explanations
- Adding or updating examples
- Keeping the tone friendly and approachable

---

Thanks for checking out Starstruck Studios. We're excited about what we're making, and we'd love for you to be part of it!
